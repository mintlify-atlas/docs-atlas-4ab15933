# Creating Items

Items are defined in `PBS/items.txt`. This includes everything from Potions and Poké Balls to evolution stones, TMs, and held items.

## Item Entry Structure

Each item is defined on a single line with comma-separated values:

```txt
ID,InternalName,Name,NamePlural,Pocket,Price,Description,FieldUse,BattleUse,Flags,Move
```

## Field Definitions

<ParamField path="ID" type="number" required>
  Unique identifier for the item. Must be sequential starting from 1.
</ParamField>

<ParamField path="InternalName" type="string" required>
  Internal code name in uppercase. Used in scripts and PBS files.
  
  Example: `POTION`, `FIRESTONE`, `POKEBALL`
</ParamField>

<ParamField path="Name" type="string" required>
  Singular display name shown to players.
</ParamField>

<ParamField path="NamePlural" type="string" required>
  Plural form of the item name.
  
  Example: `Pociones`, `Piedras Fuego`, `Poké Balls`
</ParamField>

<ParamField path="Pocket" type="number" required>
  Which bag pocket the item goes in:
  - `1` - Items (general items, stones, held items)
  - `2` - Medicine (Potions, status healers)
  - `3` - Poké Balls
  - `4` - TMs & HMs
  - `5` - Berries
  - `6` - Mail
  - `7` - Battle items
  - `8` - Key items
</ParamField>

<ParamField path="Price" type="number" required>
  Buy price at Poké Marts. Sell price is half. Use `0` for items that can't be bought.
</ParamField>

<ParamField path="Description" type="string" required>
  Item description shown in bag. Enclosed in quotes.
</ParamField>

<ParamField path="FieldUse" type="number" required>
  How the item can be used outside of battle:
  - `0` - Cannot be used
  - `1` - Use on Pokémon (most items)
  - `2` - Use directly (Repels, Escape Rope)
  - `3` - TMs/HMs (can be taught)
</ParamField>

<ParamField path="BattleUse" type="number" required>
  How the item can be used in battle:
  - `0` - Cannot be used in battle
  - `1` - Use on Pokémon (Potions, status healers)
  - `2` - Use on throw (Poké Balls)
  - `3` - Direct use
  - `4` - On enemy Pokémon
</ParamField>

<ParamField path="Flags" type="number" required>
  Special flags for the item:
  - `0` - No special flags
  - `1` - Consumable (disappears after use)
  - `2` - Can be held by Pokémon
  - `4` - Can be registered to quick access
</ParamField>

<ParamField path="Move" type="string">
  For TMs/HMs only: the internal name of the move taught.
  
  Example: `HONECLAWS`, `THUNDERBOLT`, `SURF`
</ParamField>

## Creating a New Item

<Steps>

### Step 1: Determine item type

Decide what category your item falls into:
- Medicine/consumable
- Held item
- Evolution item
- Battle item
- Key item

### Step 2: Choose appropriate pocket

Based on your item type, select the correct pocket number (1-8).

### Step 3: Set usage flags

Determine how the item can be used:
- `FieldUse`: 0 (none), 1 (on Pokémon), 2 (direct), 3 (TM)
- `BattleUse`: 0 (none), 1 (on Pokémon), 2 (ball), 3 (direct), 4 (on enemy)

### Step 4: Define the item

```txt
999,CUSTOMPOTION,Custom Potion,Custom Potions,2,500,"A special potion that restores 150 HP.",1,1,0,
```

</Steps>

<Tip>
  For items that need special behavior (like evolution stones or held items with battle effects), you may need to add code in the game's scripts under `PokemonItems` or related sections.
</Tip>

## Battle Items

Items used during battle for strategic advantage:

```txt
# X Items (stat boosters)
X Attack - Sharply raises Attack in battle
X Defense - Sharply raises Defense in battle  
X Speed - Sharply raises Speed in battle
X Sp. Atk - Sharply raises Special Attack
X Sp. Def - Sharply raises Special Defense
X Accuracy - Sharply raises Accuracy

# Guard Spec - Prevents stat reduction for 5 turns
# Dire Hit - Raises critical hit ratio
```

## Berries

Berries are special items that typically go in pocket 5:

```txt
# Healing Berries
Oran Berry - Restores 10 HP when held
Sitrus Berry - Restores 25% HP when held

# Status Berries  
Cheri Berry - Cures paralysis
Chesto Berry - Cures sleep
Pecha Berry - Cures poison
Rawst Berry - Cures burn
Aspear Berry - Cures freeze

# Type-Resist Berries
Occa Berry - Weakens super-effective Fire moves
Passho Berry - Weakens super-effective Water moves
```

<Note>
  Berry mechanics (growing, harvesting) are typically implemented through map events and scripts, not just the PBS files.
</Note>

## Key Items

Key items are plot-important and cannot be sold or discarded:

```txt
# Typically defined with pocket 8
# Examples: Bicycle, Fishing Rods, Plot Keys
# These require custom implementation in scripts
```

<Warning>
  Key items need special handling in your game's scripts to function properly. They're often tied to specific events, maps, or game progression flags.
</Warning>

## Item Icons

Each item needs a corresponding icon graphic in `Graphics/Items/`. The filename should match the internal name:

```
Graphics/Items/POTION.png
Graphics/Items/FIRESTONE.png
Graphics/Items/POKEBALL.png
```

## Testing Items

<Steps>

### Use Debug mode

Press F8 and use "Fill PC with items" or "Add item" to test.

### Check all contexts

- Bag interface
- Use on Pokémon
- Use in battle
- Held item effects

### Verify pricing

Test buying and selling at shops to ensure prices are balanced.

</Steps>
