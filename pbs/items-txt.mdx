The `items.txt` file defines all items in the game, including their properties, prices, effects, and categorization. Each item is defined on a single line in CSV format.

## File Format

Each line defines one item with comma-separated values:

```
ID,INTERNALNAME,Name,NamePlural,Pocket,Price,Description,FieldUse,BattleUse,Flags,Consumable
```

## Example Entries

```
1,REPEL,Repelente,Repelentes,1,350,"Repele Pokémon salvajes débiles en un recorrido de 100 pasos.",2,0,0,
217,POTION,Poción,Pociones,2,300,"Medicina en spray que cura heridas y restaura 20 PS de un Pokémon.",1,1,0,
264,MASTERBALL,Master Ball,Master Balls,3,0,"La Poké Ball definitiva. Atrapa a cualquier Pokémon salvaje y no falla nunca.",0,2,3,
```

## Field Definitions

<ParamField path="ID" type="Integer">
  Unique item ID number. Must be sequential starting from 1.
  
  ```
  1,REPEL,...
  2,SUPERREPEL,...
  217,POTION,...
  ```
</ParamField>

<ParamField path="InternalName" type="String">
  Internal identifier in UPPERCASE. Used in code and scripts.
  
  ```
  POTION
  MASTERBALL
  THUNDERSTONE
  ```
</ParamField>

<ParamField path="Name" type="String">
  Display name shown to players (singular form).
  
  ```
  Poción
  Master Ball
  Piedra Trueno
  ```
</ParamField>

<ParamField path="NamePlural" type="String">
  Plural form of the item name.
  
  ```
  Pociones
  Master Balls
  Piedras Trueno
  ```
</ParamField>

<ParamField path="Pocket" type="Integer">
  Bag pocket/category where the item is stored.
  
  - `0` - Key Items (cannot be sold)
  - `1` - Items (general items)
  - `2` - Medicine (healing items)
  - `3` - Poké Balls
  - `4` - TMs & HMs
  - `5` - Berries
  - `6` - Battle Items
  - `7` - Mail
  
  ```
  1  # General items pocket
  2  # Medicine pocket
  3  # Poké Balls pocket
  ```
</ParamField>

<ParamField path="Price" type="Integer">
  Purchase price at Poké Marts. Sell price is typically 50% of this.
  
  Use `0` for items that cannot be bought.
  
  ```
  300     # Potion
  1200    # Ultra Ball
  0       # Cannot buy (Master Ball)
  ```
</ParamField>

<ParamField path="Description" type="String">
  Item description shown to players. Must be in quotes.
  
  ```
  "Medicina en spray que cura heridas y restaura 20 PS de un Pokémon."
  "La Poké Ball definitiva. Atrapa a cualquier Pokémon salvaje y no falla nunca."
  ```
</ParamField>

<ParamField path="FieldUse" type="Integer">
  Usage behavior outside of battle.
  
  - `0` - Cannot use in field
  - `1` - Can use on a Pokémon
  - `2` - Can use directly (like Repel)
  - `3` - Can use on Pokémon or directly
  
  ```
  0  # Battle-only item
  1  # Use on Pokémon (Potion)
  2  # Use directly (Repel)
  ```
</ParamField>

<ParamField path="BattleUse" type="Integer">
  Usage behavior during battle.
  
  - `0` - Cannot use in battle
  - `1` - Can use on a Pokémon
  - `2` - Can use directly (Poké Balls)
  - `3` - Can use on Pokémon or directly
  
  ```
  0  # Field-only item
  1  # Use on Pokémon (Potion)
  2  # Direct use (Poké Ball)
  ```
</ParamField>

<ParamField path="Flags" type="Integer">
  Special flags for the item.
  
  - `0` - No special flags
  - `1` - Evolution stone
  - `2` - Key item (cannot toss)
  - `3` - Poké Ball
  - `4` - Multiple balls can be held
  
  ```
  0  # Normal item
  1  # Evolution item
  2  # Important key item
  ```
</ParamField>

<ParamField path="Consumable" type="String">
  Additional properties (optional, can be empty).
  
  Leave empty for most items. Used for special mechanics.
  
  ```
  
  true  # Consumed on use
  ```
</ParamField>

## Pricing Guidelines

<Accordion title="Common Item Prices">
  - **Basic healing**: 100-300 (Potions, status healers)
  - **Mid-tier healing**: 600-1200 (Super Potion, Hyper Potion)
  - **High-tier healing**: 2500+ (Max Potion, Full Restore)
  - **Poké Balls**: 200-1200 (Basic to Ultra)
  - **Evolution stones**: 2100
  - **Stat boosters**: 9800 (Vitamins)
  - **Held items**: 100-200 (most)
  - **Specialty items**: 0 (cannot buy)
</Accordion>

```
100,LIFEORB,Vidasfera,Vidasferas,1,200,"Potencia los movimientos, pero consume PS con cada ataque. Debe llevarla un Pokémon.",0,0,0,
```

- Cannot use directly (FieldUse=0, BattleUse=0)
- Must be held by Pokémon
- Effect coded in battle scripts
- General items pocket

<Note>
  **Item Effects**: The actual effects of items (HP restoration amount, stat boost values, etc.) are coded in the game scripts. The PBS file only defines the item's properties and description.
</Note>

<Note>
  **TM Items**: TMs require an additional parameter specifying which move they teach. This is appended to the end of the line after the consumable field.
</Note>

<Note>
  **Key Items**: Items with Pocket=0 or Flag=2 are key items that cannot be sold or tossed. Use these for quest items and important story objects.
</Note>
